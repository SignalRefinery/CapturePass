## 1. Environment And Config

- [ ] Production Vercel deployment is on the intended branch.
- [ ] Supabase project is the intended production project.
- [ ] Vercel env vars are set for production:
  - [ ] `NEXT_PUBLIC_SUPABASE_URL`
  - [ ] `NEXT_PUBLIC_SUPABASE_ANON_KEY`
  - [ ] `SUPABASE_SERVICE_ROLE_KEY`
  - [ ] `NEXT_PUBLIC_SITE_URL`
  - [ ] `STRIPE_SECRET_KEY`
  - [ ] `STRIPE_WEBHOOK_SECRET`
  - [ ] Stripe price IDs
  - [ ] `RESEND_API_KEY`, if using contact/shared notifications
- [ ] Supabase Auth Site URL points to the production domain.
- [ ] Supabase Auth redirect URLs include:
  - [ ] `https://yourdomain.com/auth/callback`
  - [ ] `https://yourdomain.com/update-password`
  - [ ] `https://yourdomain.com/*/login`
- [ ] Supabase SMTP sends successfully.
- [ ] `.env.local` is not committed.
